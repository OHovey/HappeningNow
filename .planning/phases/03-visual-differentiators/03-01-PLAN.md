   - `crowdScoreToColor(score: number): string` — clamp 1-10, return hex.
   - `crowdScoreToLabel(score: number): string` — return "Low season" (1-3), "Moderate" (4-6), "Busy" (7-8), "Peak crowds" (9-10).
   - `estimateTouristVolume(score: number): string` — rough human-readable estimate (e.g. "Very few tourists", "Moderate flow", "Heavy crowds", "Extremely busy").

2. Create `src/lib/map/heatmap.ts` with:
   - `DestinationWithCoords` type: `{ id, name, slug, lng, lat, crowd_data, weather_data }` (or import from types.ts if added there).
   - `buildCrowdHeatmapSource(destinations, month): GeoJSON.FeatureCollection` — maps each destination to a Point feature with `crowd_score`, `name`, `slug`, `tourist_volume` properties. Use `crowd_data?.[String(month)] ?? 5` for the score.
   - `crowdHeatmapLayer: LayerSpecification` — MapLibre heatmap layer config with:
     - `id: 'crowd-heatmap'`, `source: 'crowd-heatmap'`, `type: 'heatmap'`
     - `heatmap-weight`: interpolate linear on `crowd_score` from 1→0 to 10→1
     - `heatmap-intensity`: interpolate by zoom (0→1, 9→3)
     - `heatmap-color`: 6-stop interpolation on heatmap-density from transparent purple through blue, amber, deep orange to red (matching CROWD_COLORS palette). Per user decision: cool-to-warm.
     - `heatmap-radius`: interpolate by zoom (0→20, 5→40, 9→60) for regional blob granularity per user decision.
     - `heatmap-opacity`: 0.6 for subtle background wash per user decision.
   - Export `toggleHeatmapVisibility(map, visible)` — calls `map.setLayoutProperty('crowd-heatmap', 'visibility', visible ? 'visible' : 'none')`.

3. Fix weather_data type mismatch in `src/lib/supabase/types.ts`:
   - Change `Destination.weather_data` type from `Record<string, { temp: number; rain: number; sunshine: number }>` to `Record<string, { temp_c: number; rain_days: number; sunshine_hours: number }>` to match actual seed data keys in the database.

4. Add `DestinationWithCoords` interface to `src/lib/supabase/types.ts`:
   - `export interface DestinationWithCoords extends Omit<Destination, 'location'> { lng: number; lat: number; }`

5. Add destination query functions to `src/lib/supabase/queries.ts`:
   - `getAllDestinationsWithCoords(): Promise<DestinationWithCoords[]>` — new RPC `get_destinations_with_coords` extracting ST_X/ST_Y from PostGIS geometry (following existing `get_event_with_coords` pattern). If RPC doesn't exist yet, create a SQL migration file `supabase/migrations/get_destinations_with_coords.sql` with the RPC.
   - `getDestinationBySlug(slug): Promise<DestinationWithCoords | null>` — single destination fetch.
   - `getAllDestinationSlugs(): Promise<string[]>` — for generateStaticParams.
   - `getEventsByDestination(destination): Promise<Event[]>` — events matching destination region/country, for calendar grid.
