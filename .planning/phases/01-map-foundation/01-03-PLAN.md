  <action>
Create `scripts/seed.ts` that:

1. Reads all four JSON seed files
2. Connects to Supabase using `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` (service role for write access — add to .env.local.example)
3. Clears existing data (DELETE FROM events, destinations, migration_routes — with confirmation prompt or --force flag)
4. Inserts festivals and wildlife events into `events` table:
   - Convert longitude/latitude to PostGIS point using Supabase RPC or raw SQL: `ST_SetSRID(ST_MakePoint(lng, lat), 4326)`
   - Use batch inserts (50 rows at a time) to avoid timeouts
   - Generate slug from name if not provided (lowercase, hyphenated)
5. Inserts destinations into `destinations` table with same point conversion
6. Inserts migration routes into `migration_routes` table:
   - Convert coordinate array to PostGIS LineString: `ST_SetSRID(ST_MakeLine(array of ST_MakePoint), 4326)` or use raw SQL with `ST_GeomFromGeoJSON`
7. Logs progress: "Inserted X/Y festivals..." with counts
8. Verifies counts match expectations after insert

Add to package.json scripts: `"seed": "npx tsx scripts/seed.ts"`

Add `SUPABASE_SERVICE_ROLE_KEY` to `.env.local.example` with source comment:
"Supabase Dashboard -> Project Settings -> API -> service_role (secret)"

Install tsx if not present: `npm install -D tsx`

**IMPORTANT:** The seed script must handle PostGIS geometry insertion correctly. Since Supabase JS client doesn't natively support geometry types, use `.rpc()` calls or raw SQL via `supabase.rpc('exec_sql', { query: ... })`. Alternative: create a Supabase function `insert_event_with_point(...)` that accepts lng/lat and internally uses ST_MakePoint. Choose the simplest working approach.
