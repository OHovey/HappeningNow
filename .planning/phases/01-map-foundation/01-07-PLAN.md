Create `src/lib/convertkit.ts`:
- Kit API v4 base URL: `https://api.kit.com/v4`
- Helper function for headers: `{ 'Content-Type': 'application/json', 'X-Kit-Api-Key': process.env.KIT_API_KEY! }`

Export `createSubscriber(email: string): Promise<{ id: number; email: string }>`:
- POST to `${BASE}/subscribers`
- Body: `{ email_address: email, state: 'active' }`
- Return `subscriber` from response
- Handle errors: if 422 (already exists), try to fetch existing subscriber
- If KIT_API_KEY not set, throw descriptive error

Export `tagSubscriber(subscriberId: number, tagId: number): Promise<void>`:
- POST to `${BASE}/tags/${tagId}/subscribers/${subscriberId}`
- No body needed (just the URL path identifies the relationship)
- IMPORTANT per RESEARCH.md Pitfall 7: subscriber MUST exist before tagging. Always call createSubscriber first.

Export `TAG_IDS` constant (to be configured):
- `{ festivals: 0, wildlife: 0 }` — placeholder IDs, replaced with real Kit tag IDs from dashboard
- Add comment: "Replace with real tag IDs from Kit Dashboard -> Subscribers -> Tags"

Create `src/app/api/subscribe/route.ts`:
- Export async `POST(request: Request)` handler
- Parse JSON body: `{ email, interests: string[], eventCategory: string }`
- Validate email format (simple regex, reject if invalid — return 400)
- Validate interests is an array (default to empty if missing)
- Call `createSubscriber(email)`
- For each interest in `interests` plus the `eventCategory`:
  - Look up tag ID from `TAG_IDS`
  - Call `tagSubscriber(subscriber.id, tagId)`
- Return `NextResponse.json({ success: true })` on success
- Return `NextResponse.json({ error: message }, { status: 500 })` on failure
- If KIT_API_KEY not set, return 503 with message "Email service not configured"

Create `tests/api/subscribe.test.ts`:
- Mock `fetch` globally to intercept Kit API calls
- Test: valid email creates subscriber and returns success
- Test: invalid email returns 400
- Test: interests array triggers correct tag calls
- Test: eventCategory auto-tags subscriber
- Test: Kit API error returns 500 to client
- Test: missing KIT_API_KEY returns 503
