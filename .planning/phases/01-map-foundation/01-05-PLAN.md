Create `src/lib/map/filters.ts`:
- Export `filterGeoJSON(geojson: GeoJSON.FeatureCollection, month: number, categories: string[]): GeoJSON.FeatureCollection`
  - Filters features where: `start_month <= month <= end_month` (handle wrap-around: if start_month > end_month, event spans year boundary e.g., Dec-Feb)
  - Filters features where: `category` is in the `categories` array
  - Returns a new FeatureCollection with only matching features
  - This is the approach for ~600 events — update the source data so clustering counts are correct (see RESEARCH.md Pitfall 2)
- Export `buildMonthFilter(month: number)` that returns a MapLibre filter expression (for cases where source-level filtering is not used)
- Export `buildCategoryFilter(categories: string[])` — same
- Export `getCurrentMonth(): number` — returns current month (1-12) for default view

**Why source-level filtering (filterGeoJSON + setData) instead of layer-level setFilter:** Per RESEARCH.md Pitfall 2, clustering operates at the source level. Using setFilter on layers would show filtered markers but cluster counts would include hidden events. With ~600 events, setData on the filtered set is fast and keeps clusters accurate.
