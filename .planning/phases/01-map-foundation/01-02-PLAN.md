  <name>Task 1: Create PostGIS schema with spatial tables and indexes</name>
  <files>
    supabase/schema.sql

  <action>
Create `supabase/schema.sql` with the following tables exactly as specified in RESEARCH.md:

**events table:**
- `id` UUID DEFAULT gen_random_uuid() PRIMARY KEY
- `name` TEXT NOT NULL
- `slug` TEXT UNIQUE NOT NULL
- `category` TEXT NOT NULL CHECK (category IN ('festival', 'wildlife'))
- `description` TEXT
- `image_url` TEXT
- `start_month` INTEGER NOT NULL CHECK (start_month BETWEEN 1 AND 12)
- `end_month` INTEGER NOT NULL CHECK (end_month BETWEEN 1 AND 12)
- `location` geometry(Point, 4326) NOT NULL — use geometry NOT geography (5-10x faster; locked decision)
- `country` TEXT
- `region` TEXT
- `scale` INTEGER DEFAULT 5 CHECK (scale BETWEEN 1 AND 10)
- `crowd_level` TEXT CHECK (crowd_level IN ('quiet', 'moderate', 'busy'))
- `booking_destination_id` TEXT
- `getyourguide_location_id` TEXT
- `created_at` TIMESTAMPTZ DEFAULT now()

Indexes on events:
- `events_location_idx` USING GIST (location)
- `events_category_idx` ON (category)
- `events_month_idx` ON (start_month, end_month)

**destinations table:**
- `id` UUID DEFAULT gen_random_uuid() PRIMARY KEY
- `name` TEXT NOT NULL
- `slug` TEXT UNIQUE NOT NULL
- `country` TEXT NOT NULL
- `region` TEXT
- `location` geometry(Point, 4326) NOT NULL
- `crowd_data` JSONB — monthly crowd scores (1-10) per month
- `weather_data` JSONB — monthly temp/rain/sunshine per month
- `created_at` TIMESTAMPTZ DEFAULT now()

Index: `destinations_location_idx` USING GIST (location)

**migration_routes table:**
- `id` UUID DEFAULT gen_random_uuid() PRIMARY KEY
- `species` TEXT NOT NULL
- `name` TEXT NOT NULL
- `slug` TEXT UNIQUE NOT NULL
- `route` geometry(LineString, 4326) NOT NULL
- `peak_months` INTEGER[] NOT NULL
- `description` TEXT
- `image_url` TEXT
- `created_at` TIMESTAMPTZ DEFAULT now()

Index: `migration_routes_route_idx` USING GIST (route)

Enable PostGIS at the top: `CREATE EXTENSION IF NOT EXISTS postgis;`

Create `supabase/functions/get_events_geojson.sql`:
- `CREATE OR REPLACE FUNCTION get_events_geojson()` that returns a GeoJSON FeatureCollection
- Include all event properties needed by the map: id, name, slug, category, start_month, end_month, scale, crowd_level, image_url, description, country, region, booking_destination_id, getyourguide_location_id
- Use `ST_AsGeoJSON(location)::json` for geometry conversion
- Handle empty table with COALESCE to empty array

Add a comment at the top of schema.sql: "Run this in Supabase SQL Editor (Dashboard -> SQL Editor -> New Query)"
