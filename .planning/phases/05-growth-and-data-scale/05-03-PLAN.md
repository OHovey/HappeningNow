  <action>
1. Create `scripts/scrape-wikipedia.ts` — one-time/manually-triggered script per user decision:
   - Install cheerio if not present: `npm install cheerio`
   - Target a small curated list of well-structured Wikipedia pages (per research pitfall 5):
     - "List_of_music_festivals" (has table with Name, Location, Country, Genre)
     - "List_of_food_festivals" (has Name, Location, Date columns)
     - "List_of_film_festivals" (has Name, Location, Founded columns)
     - 2-3 more pages as appropriate
   - For each page:
     a) Fetch HTML via MediaWiki API: `https://en.wikipedia.org/w/api.php?action=parse&page={PAGE}&prop=text&format=json`
     b) Set User-Agent: 'HappeningNow.travel/1.0 seed-script (contact: admin@happeningnow.travel)'
     c) Parse tables with cheerio: load HTML, find `table.wikitable tr`, extract cells
     d) Per-page column mapping object (column indexes vary by page — define explicitly)
     e) For each row: extract name, location string, country, optional dates
   - Geocode each location with Nominatim (NOT Photon — Nominatim is appropriate for batch single lookups):
     - `https://nominatim.openstreetmap.org/search?q={location},{country}&format=json&limit=1`
     - CRITICAL: 1 second delay between requests (Nominatim usage policy)
     - Set User-Agent header per Nominatim policy
     - Extract lat, lon from first result
   - Output: JSON array of event objects matching events table schema:
     - { name, slug (slugified name), category: 'festival', description: null, image_url: null, start_month: null, end_month: null, lng, lat, country, region, scale: 5 (default), crowd_level: null }
   - Write output to `src/data/seed/wikipedia-festivals.json`
   - Support --dry-run flag: parse and geocode but don't write file
   - Log progress: "Scraping {page}... Found {n} rows. Geocoded {m}/{n}."
   - Skip rows where geocoding fails (log warning)
   - Add npm script: `"scrape-wikipedia": "tsx scripts/scrape-wikipedia.ts"`
   - Per user decision: this is a MINIMAL one-time script, NOT a reusable pipeline framework. Keep it simple.
  </action>
