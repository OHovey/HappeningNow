1. Create `supabase/functions/search_events_nearby.sql` — a Supabase RPC function that:
   - Accepts parameters: user_lng float, user_lat float, radius_meters float, start_m int DEFAULT NULL, end_m int DEFAULT NULL, filter_category text DEFAULT NULL
   - Returns table with: id, name, slug, category, description, image_url, start_month, end_month, lng (ST_X), lat (ST_Y), country, region, scale, crowd_level, booking_destination_id, getyourguide_location_id, distance_meters
   - Uses `ST_DWithin(e.location::geography, ST_SetSRID(ST_MakePoint(user_lng, user_lat), 4326)::geography, radius_meters)` — CRITICAL: must cast to ::geography for meter-based distance, the events table uses geometry(Point, 4326) NOT geography
   - Month overlap logic with CASE for wrap-around (same pattern as get_events_bbox.sql): when start_month <= end_month use simple range check, else handle Dec-Feb wrap
   - Orders by distance_meters ASC
   - Run this SQL in the Supabase dashboard or via `supabase db push`

2. Create `src/app/api/geocode/route.ts` — Photon proxy GET route:
   - Extract `q` from searchParams, return empty features array if q missing or < 2 chars
   - Use in-memory Map cache with 1-hour TTL (acceptable for serverless — helps within session)
   - Fetch from `https://photon.komoot.io/api/` with params: q, limit=5, lang=en, layer=city,state,country
   - Set User-Agent header: 'HappeningNow.travel/1.0'
   - Return Photon GeoJSON FeatureCollection directly
   - On Photon error, return `{ features: [] }` with status 502
   - DO NOT use Nominatim for autocomplete — it is forbidden by their usage policy. Photon is designed for search-as-you-type.

   - Return JSON array of events with distance_meters included
   - On validation error return 400 with structured error, on Supabase error return 500
   - Follow the existing pattern in src/app/api/events/route.ts for Zod validation style

1. Create `src/lib/scoring.ts` with three exported functions:

   a) `worthTheTripScore(event)` — "worth the trip" ranking score:
      - Input: { scale: number, crowd_level: string | null, distance_meters: number }
      - Crowd penalty map: quiet=1.0, moderate=0.7, busy=0.4 (null defaults to moderate)
      - Formula: (scale^2 * crowdFactor) / log2(distanceKm + 1) — scale squared makes uniqueness dominant per user decision
      - Min distance 1km to avoid division by zero
      - Returns number (higher = better, hidden from users per user decision)

   b) `estimateTravelTime(distanceMeters)` — travel time string:
      - Haversine straight-line at ~60km/h average per user decision
      - < 1 hour: "Xmin drive", >= 1 hour: "X.Xh drive"

   c) `getIndicatorTags(event)` — array of indicator tag strings:
      - scale >= 8: "Highly Unique", scale >= 6: "Unique"
      - crowd_level === 'quiet': "Low Crowds"
      - Always includes travel time estimate
      - These are the quick indicator tags on each card per user decision
