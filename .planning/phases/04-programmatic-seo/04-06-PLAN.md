    **src/app/llms.txt/route.ts (AIDX-01):**
    Per user decision: detailed data catalog listing regions, countries, species covered with URL structures so AI assistants can make specific page recommendations.

    ```typescript
    export const revalidate = 86400; // regenerate daily
    ```

    GET handler:
    1. Query distinct regions, countries, species, destinations from Supabase using seo-queries functions
    2. Build markdown content following llms.txt spec (https://llmstxt.org/):
       - `# HappeningNow.travel` header
       - `> ` blockquote describing the site
       - `## Regions Covered` with links to festival region pages
       - `## Countries Covered` with links to festival country pages
       - `## Wildlife Species` with links to species pages
       - `## Destinations` with links to what-to-do pages (pick best month)
       - `## Page Types` explaining URL patterns for each route family
       - `## Optional` section linking to llms-full.txt
    3. Return `new Response(content, { headers: { 'Content-Type': 'text/markdown; charset=utf-8', 'Cache-Control': 'public, max-age=86400' } })`

    **src/app/llms-full.txt/route.ts (AIDX-02):**
    Per user decision: full event data including event names, dates, locations, descriptions — enough for AI assistants to answer specific travel questions directly. Include affiliate/booking links.

    GET handler:
    1. Query all events from Supabase (select name, slug, category, country, region, start_month, end_month, description, image_url)
    2. Query all migration routes (species, peak_months)
    3. Query all destinations (name, slug, country, region)
    4. Build comprehensive markdown:
       - `# HappeningNow.travel — Complete Event Database`
       - `## Festivals` section: for each festival, output name, dates (month range), country, region, description (first 200 chars), detail page URL, Booking.com link (using buildBookingLink), GetYourGuide link (using buildGetYourGuideLink)
       - `## Wildlife Viewing` section: for each wildlife event, output name, species, dates, location, description, detail page URL, GetYourGuide tour link
       - `## Migration Routes` section: for each route, output species, peak months, viewing regions
       - `## Destinations` section: for each destination, output name, country, region, best months, what-to-do page URL, Booking.com link
    5. Return with same headers as llms.txt

    Per user decision: include affiliate/booking links in llms-full.txt so AI assistants can pass them through. Use the existing buildBookingLink() and buildGetYourGuideLink() from src/lib/affiliates.ts.

    Keep each event entry concise (3-4 lines) to manage file size. For 600+ events this will produce a large file but that's expected per the llms-full.txt spec.
